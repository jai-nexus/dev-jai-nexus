### Enforcement mechanics and blocked-path audit

- Blocked prefixes, exactly ten: `.tmp/`, `.vercel/`, `.vscode/`,
  `node_modules/`, `portal/node_modules/`, `portal/.next/`,
  `portal/.vercel/`, `portal/out/`, `portal/generated/`, and `vault/`.
- Blocked exact paths, exactly ten: `.env`, `.env.local`, `portal/.env`,
  `portal/.env.local`, `portal/.env.vercel.local`,
  `portal/.env.vercel.production`, `portal/dev.db`, `tmp.sql`,
  `Makefile.tmp`, and `portal/tsconfig.tsbuildinfo`.
- Global block: every changed path containing the substring `.env` fails,
  independently of the exact and prefix lists.
- Role extraction: the workflow reads the first case-insensitive line matching
  `Role: <value>` anywhere in the PR body. It trims the value, splits
  colon-bearing input, canonicalizes a `JAI` plus `DEV` prefix, uppercases the
  remaining role tail, and requires the configured `JAI::DEV::` prefix.
- Effective Verifier bypass: `rolemap.verifierBypassAllowedRoles ?? true`
  evaluates to `true` because the field is absent. A normalized
  `JAI::DEV::VERIFIER` skips ordinary `allowedRoles` checks for every
  non-blocked path. It does not skip blocked-path checks or aggregate Evidence.
- Aggregate Evidence: `evidenceRequired` begins false and becomes true when
  any matched rule has `evidenceRequired: true`; the workflow then requires a
  non-empty configured Evidence section. DEFAULT sets the workflow fallback
  to false. The current A8 packet and Draft-PR contract independently require
  meaningful Evidence despite that DEFAULT result.
- These are current static mechanics and gaps. A8 does not authorize a
  rolemap, workflow, PR-template, or path-policy repair.
