## 7. Design brief objective

The future static UI design brief should describe a cross-repo state surface that helps an operator scan evidence-summary posture without replacing source truth, mutating Program Plans, accepting closeouts, creating receipts, activating Agent PR factory behavior, or opening gates.

The design objective is to make the difference between summary, source, warning, receipt, route note, and CONTROL_THREAD decision posture visible before any implementation-readiness route.

Design-brief planning does not create component behavior, page behavior, route behavior, API/DB payloads, live state, runtime behavior, authority, or gate behavior.
