# Acceptance Criteria Checklist

## Purpose

Standardize acceptance review so completion claims stay falsifiable.

## When to use

Use when a motion or bounded seam includes named acceptance checks.

## Required inputs

- acceptance list
- implemented outputs
- validation evidence

## Expected outputs

- explicit satisfied/not-satisfied mapping

## Checklist

- list each acceptance item
- state whether it passed
- tie each pass to observed output or evidence
- state any residual gaps

## Hard boundaries

- do not merge multiple criteria into one vague pass claim
- do not mark future work as satisfied now

## Related canon

- motion execution evidence patterns
