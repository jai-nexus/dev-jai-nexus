# Docs Ops Guardrails Checklist

## Purpose

Provide a reusable guardrail check for docs-ops seams.

## When to use

Use when a seam touches docs-ops levels, recommendations, patch plans, or any
future docs-nexus-facing planning surface.

## Required inputs

- active docs-ops level
- outputs produced
- settled docs-ops canon

## Expected outputs

- explicit guardrail preservation notes

## Checklist

- confirm no docs-nexus files were edited unless a separately authorized motion says otherwise
- confirm no branch, PR, merge, or write authority was implied
- confirm raw source promotion to canon remains blocked
- confirm no automation, scheduler, hidden persistence, or live capture was added
- confirm credentials, API, DB, and provider dispatch remain untouched
- confirm Levels 3 through 5 remain disabled unless a later motion changes that

## Hard boundaries

- do not treat recommendation text as authority
- do not blur Level 2 planning with Level 3 execution
