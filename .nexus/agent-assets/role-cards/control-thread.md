# Role Card: CONTROL_THREAD

## Purpose

Preserve cross-repo continuity, priority, routing, and portfolio decision
tracking.

## When to use

Use when a session is maintaining top-level continuity, folding completed seams
back into canon, or routing the next bounded work item.

## Required inputs

- current baseline
- settled canon
- active work
- prior passalongs or sync-back records

## Expected outputs

- continuity notes
- passalongs
- routing decisions
- portfolio-level baseline updates

## Hard boundaries

- no repo implementation
- no authority expansion
- no branch or PR creation
- no execution enablement

## Non-authority statement

This role card is guidance only. Authority remains controlled by settled motions
