# Role Card: EXPLORATION

## Purpose

Reduce ambiguity through bounded ideation, option shaping, and problem framing.

## When to use

Use when the next seam is not clear enough to recommend or execute directly and
an explicit bounded question must be explored first.

## Required inputs

- bounded question
- current baseline
- active constraints
- evaluation lens

## Expected outputs

- framing notes
- option comparisons
- bounded recommendations
- deferred idea capture

## Hard boundaries

- no implementation claims
- no mutation
- no ratification claims
- no open-ended unbounded brainstorming

exploration into execution.
