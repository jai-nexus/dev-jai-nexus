# JAI Agent Assets Library v0

## Purpose

This library provides static, repo-native operating assets for future JAI Agent
planning, review, verification, passalong, repo-routing, and docs-ops work.

These assets standardize reusable prompts, checklists, role cards, and review
materials so sessions can start from consistent scaffolding instead of
reconstructing operating structure each time.

## When to use

Use this library when a session needs:

- role-specific operating guidance
- reusable prompt or artifact templates
- reusable passalong or repo-routing materials
- reusable verification, operator, risk, or acceptance checklists
- reusable docs-ops recommendation or Level 2 patch-plan planning materials

## Asset taxonomy

1. role cards
2. prompt templates
3. passalong templates
4. repo-routing templates
5. verification checklists
6. docs-ops recommendation template
7. Level 2 patch-plan template
8. operator review checklist
9. risk register template
10. acceptance criteria template

## Relationship to canon

- Workflow-role assets relate to motion-0173 and the settled workflow-role taxonomy.
- Docs-ops assets relate to motions 0174, 0175, and 0176.
- Authority remains governed by motions and explicit operator or control-thread decisions.

## Hard boundary

These assets are static operating material only.

- They are not executable policy.
- They do not grant authority.
- They do not replace workflow-role canon.
- They do not replace the docs-ops authority ladder.
- They do not activate docs-ops Level 3, Level 4, or Level 5.
- They do not mutate other repos.
