# Repo Routing Target Template

## Purpose

Provide a reusable repo-routing package structure aligned to motion-0173 canon.

## When to use

Use when a bounded seam is being routed into repo-local work or structured
exploration.

## Required inputs

- repo
- branch
- motion or seam
- baseline
- task
- allowed files/surfaces
- blocked files/surfaces
- acceptance criteria
- validation commands
- expected return fields
- passalong destination

## Expected outputs

- a bounded repo-routing target package

## Template

```md
## Repo

## Branch

## Motion or seam

## Baseline

## Task

## Allowed files/surfaces

## Blocked files/surfaces

## Acceptance criteria

## Validation commands

## Expected return fields

## Passalong destination
```

## Hard boundaries

- do not imply authority beyond the active motion
- do not use as an automation trigger

## Related canon

- motion-0173

## Non-authority statement

This template standardizes routing text only and does not grant write, branch,
PR, or execution authority.
