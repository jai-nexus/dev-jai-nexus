# Corpus V2 Pre-Opening Operator Checklist

## Purpose

Give `CONTROL_THREAD` a concise operator-facing checklist for what must be true
before a future Corpus V2 opening motion can even be proposed.

This is pre-opening only. It does not open Corpus V2.

## Operator-facing checklist

Before any future opening proposal, confirm:

- readiness gates reviewed
- static readiness data model current
- readiness drilldown surface current
- sandbox boundary defined
- agent drafting prototype designed or demonstrated
- agent voting prototype designed or demonstrated
- authority boundaries reviewed
- Corpus V1 immutability preserved
- numbering reset protocol drafted

## Required evidence before opening proposal

The future opening proposal should have:

- current readiness checklist review
- current static readiness model review
- current drilldown surface review
- current sandbox boundary review
- current drafting design review or bounded demonstration
- current voting and ratification design review or bounded demonstration
- current authority-boundary review
- current Corpus V1 immutability confirmation
- current opening protocol draft review

## What must still be explicitly motionized later

This pre-opening checklist does not replace later motions.

A future opening still requires explicit motionization for:

- actual Corpus V2 opening
- any numbering reset action
- any authority expansion
- any live drafting activation
- any live voting activation
- any autonomous ratification activation

## Non-goals

This checklist does not:

- open Corpus V2
- reset numbering
- activate live drafting or voting
- add automation
- add enforcement

## Authority boundary

This checklist preserves the settled authority boundary:

- no provider or model calls
- no Agent runtime execution
- no branch-write authority
- no PR-creation authority
- no merge authority
- no scheduler
- no automation
- no API or DB mutation
