## Purpose

This checklist gives operators a manual review path for the static draft-review prototype.

It is human-gated, read-only, and non-authoritative. It does not open Corpus V2, reset numbering, or enforce runtime behavior.

## Checklist

- Draft identity is present and clearly marked as a placeholder or simulated draft.
- Target repo and target domain are clear.
- Problem statement is clear and reviewable.
- Proposed scope is bounded and specific.
- Non-goals are explicit.
- Role/lens contributors are identified.
- Canonical governance voter path is not confused with role/lens contributors.
- Validation expectations are present.
- Human review requirement is explicit.
- Authority boundary is explicit.
- No live, provider, or runtime claims are made.
- Fixture output is not canon unless later motionized.

## Review Notes

Operators should confirm that the draft fixture remains:

- static
- simulated
- review-only
- not canon
- no authority
- Corpus V2 not open

## Non-Goals

- automation
- runtime enforcement
- live agent drafting
- live voting
- fixture promotion to canon

## Authority Boundary

- No provider/model calls
- No runtime execution
- No branch-write authority
- No PR-creation authority
- No merge authority
- No scheduler
- No automation
- No API/DB mutation
- No hidden persistence
