# Agent Governance Sandbox Boundary v0

## Purpose

Define the future no-provider and no-authority sandbox boundary for
agent-operable motion drafting and voting experiments.

## Sandbox boundary

Sandbox here means:

- static or local fixtures only
- no provider or model dispatch
- no live agent runtime
- no branch write
- no PR creation
- no merge authority
- no scheduler
- no autonomous ratification

Sandbox output is not canon unless it is later motionized.

## Allowed future sandbox materials

A future sandbox may test:

- draft motion package shape
- canonical voter identity shape
- role or lens contributions
- validation evidence shape
- pass or fail simulated ratification traces

These materials remain fixtures or local test artifacts, not live governance.

## Disallowed authority

Sandbox work must not:

- dispatch providers or models
- execute live agents
- write branches
- create PRs
- merge changes
- schedule tasks
- trigger automation
- ratify motions autonomously

## Fixture posture

- fixtures may illustrate structure
- fixtures may illustrate evidence shapes
- fixtures may illustrate simulated pass or fail traces
- fixtures must remain visibly non-live and non-authorizing

## Relationship to Corpus V2 readiness

This boundary supports future readiness work by clarifying what kinds of local
experiments may occur without implying live agent operability.

## Relationship to drafting and voting designs

This boundary complements:

- `agent-motion-drafting-design-v0.md`
- `agent-voting-ratification-design-v0.md`

Those artifacts define future design posture. This artifact defines the safe
fixture-only sandbox boundary around future experiments.

## Non-goals

This boundary does not:

- enable live drafting
- enable live voting
- enable autonomous ratification
- open Corpus V2
- reset numbering
- grant any authority

## Authority boundary

This sandbox boundary preserves the settled authority boundary:

- no provider or model calls
- no Agent runtime execution
- no branch-write authority
- no PR-creation authority
- no merge authority
- no scheduler
- no automation
- no API or DB mutation
- no hidden persistence
