# Proposal: Agent Motion Drafting Design v0

## Purpose

Define the future design for JAI Agents drafting motions internally to
`dev.jai.nexus`.

## Scope

- add canon defining future draft states
- define drafting roles and human boundaries
- define required draft outputs
- preserve the distinction between governance voters, execution or lens
  participants, and human intervention
- keep the seam design-only with no live implementation

## Non-goals

- no live drafting
- no provider or model calls
- no runtime execution
- no Corpus V2 opening
- no numbering reset
- no authority expansion
