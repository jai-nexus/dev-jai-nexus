# Proposal: Corpus V2 Machine-Checkable Gate Model v0

## Purpose

Model which Corpus V2 readiness gates can eventually become deterministic
checks without implementing enforcement today.

## Scope

- add canon describing future gate checkability categories
- define gate evidence fields and a gate table
- relate the model to the readiness checklist, `validate-motion`, and
  `build-motion-snapshot`
- keep the seam design-only and non-enforcing

## Non-goals

- no enforcement
- no automation
- no Corpus V2 opening
- no numbering reset
- no live drafting, voting, or ratification enablement
