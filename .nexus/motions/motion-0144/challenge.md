# Challenge: JAI Workflow Composition Canon v0

**Motion:** motion-0144

---

## Risks

- **R-1: The workflow composition doc could be read as a runtime controller or
  session dispatcher** — a document that defines transition classes and state
  contracts resembles a workflow engine specification. Implementers or future
  sessions might treat it as an executable configuration rather than a
  documentation artifact.
  — mitigated: the definition explicitly states "cross-layer session sequence,
  not an execution mechanism." The boundary rules section explicitly prohibits
  turning the workflow chain into a controller or dispatcher. The document
  describes how sessions relate; it does not dispatch them. Defect Trap 1 in
  execution.md reinforces this for the implementer.

- **R-2: The transition matrix could be misread as an exhaustive enforced
  ruleset** — listing four transition classes (normal, optional, discouraged,
  prohibited) might imply that any transition not listed is automatically
  prohibited, or that the matrix is enforced at runtime.
  — mitigated: the template is under `.nexus/docs/templates/`, not any schema
  or enforcement directory. The motion explicitly prohibits enforcement
  infrastructure and runtime gates. The transition matrix documents known patterns;
  it is not exhaustive by default.

- **R-3: The state-carrying contract could be confused with an automated state
  transfer protocol** — specifying which state buckets must travel across
  boundaries could suggest that the system automatically packages and transfers
  this state, rather than requiring human-authored passalong documents.
  — mitigated: the state-carrying section explicitly references passalong-schema.md
  v1.1 as the mechanism. Passalong documents are authored by the session, not
  generated by a system. The contract specifies what must be included; the
  author provides it.

- **R-4: "State may be compressed but not silently dropped" could be interpreted
  loosely** — without a definition of "silently dropped," sessions might drop
  significant state items and claim they were merely compressed.
  — mitigated: the failure-state guidance section uses a reconstructability test
  rather than an intent-based definition. An omission is treated as a silent drop
  when the receiving session cannot detect it from the passalong — that is, when
  no gap is visible and the session would have to infer what is missing. If the
  passalong documents the compression or omission, the receiving session can
  identify the gap and request state-bucket fill. The recovery path is the same
  in either case; the framing avoids assigning blame and focuses on detection
  and recovery.

- **R-5: The example could prescribe a specific loop structure as the only valid
  workflow** — showing specific paths (normal loop, EXPLORATION insertion, deferred)
  might lead sessions to believe those are the only valid compositions.
  — mitigated: the example is marked explicitly as illustrative only, placed under
  `.nexus/docs/examples/`, and uses placeholder values. The composition model doc
  introduces the transition classes as a framework, not as an exhaustive list of
  the only valid paths.

- **R-6: This motion could be read as re-opening or superseding the individual
  mode canon docs** — defining how modes compose might seem to override or extend
  what each mode doc says about its own behavior.
  — mitigated: the proposal explicitly states the motion does not modify any
  existing `.nexus/docs/` documents. The composition doc cross-references the mode
  docs but does not replace or extend them. If the composition doc ever contradicts
  a mode doc, the mode doc is authoritative.
