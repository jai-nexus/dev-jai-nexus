# Proposal: Sandbox Validator Evidence Record v0

## Purpose

Document what the sandbox validator checks and what a passing run means.

## Scope

- add a canon evidence record for `node portal/scripts/validate-sandbox-fixtures.mjs`
- record fixture files checked, structural expectations, canonical voter checks, label checks, and human-boundary checks
- state clearly what PASS does and does not mean

## Non-goals

- no runtime enforcement
- no authority grant
- no Corpus V2 opening
- no live agent activation
