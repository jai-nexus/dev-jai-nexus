# Proposal: Corpus V2 Readiness Checklist v0

## Purpose

Create a read-only Corpus V2 readiness checklist and tracker before any Corpus
V2 opening work advances.

## Scope

- inspect existing corpus canon for model, closeout, readiness, schema hygiene,
  and archive posture
- add `corpus-v2-readiness-checklist.md` as an inspectable tracker
- represent readiness gates from `motion-0197` with explicit statuses
- record current evidence, missing work before V2, and authority notes
- add a docs/canon-only motion package for `motion-0201`
- refresh the bundled motion snapshot through `motion-0201`

## Non-goals

- no opening of Corpus V2
- no numbering reset
- no mutation of historical outcomes, statuses, timestamps, or vote values
- no rewriting of older motion packages
- no live agent drafting, voting, or ratification enablement
- no runtime execution, provider/model calls, branch-write, PR, merge,
  scheduler, automation, API/DB mutation, or hidden persistence authority
